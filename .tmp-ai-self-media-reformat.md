## 2. 成熟脚本示例

### 三个反常识 Prompt 技巧

**状态**：接近可发  
**平台**：小红书 / 抖音  
**内容类型**：工具干货

**一句话观点**  
Google 最新研究说明，一个看起来很蠢的 Prompt 小动作，可能让非推理模型的效果大幅提升。

#### Hook

Google 最新论文发现，只要做一个简单动作，AI 的准确率就可能从 21% 飙升到 97%。

#### 结构

1. 抛出反常识数据，先打断用户惯性认知  
2. 给出 3 个具体技巧，不讲抽象大道理  
3. 每个技巧都补一个真实例子，降低理解门槛  
4. 结尾引导评论区互动，提高停留和互动率

#### 配图 / 画面建议

- 开头放 `21% → 97%` 对比图
- 技巧一放 Google 论文截图
- 技巧二做左右对比图：普通回答 vs “医院版”回答
- 技巧三放 Reddit 帖子截图，高亮 `95% understanding`

#### CTA

提示词我放评论区，直接复制即可。你还有什么好用技巧，也可以在评论区告诉我。

## 3. 观点拆解示例

### Clawdbot 思考

**内容定位**：观点长文 / 商业判断  

#### 核心结论

- Clawdbot 更像概念展示，不是稳定成立的产品形态
- 很多人关注的是“手机能不能用、能不能连聊天软件”
- 真正决定上限的，是底层模型能力和 token 成本

#### 建议的写法

- 不要一上来就整段长文
- 先用 3 句结论打头
- 每个结论下面只放 1 个例子
- 最后再拉高到“Agent 什么时候才会成为现象级产品”

#### 更适合的结构

1. 为什么这次爆红容易让人误判  
2. 它真正解决了什么，没有解决什么  
3. 什么条件成熟后，类似产品才可能真的成立

---

### 张雪峰案例

**内容定位**：商业模式 / 系统化观点  

#### 核心结论

- 很多高客单非标生意，本质上卖的是老板本人的判断和背书
- 生意看起来成立，可能只是老板个人能力在硬撑
- AI Agent 的真正价值，不只是提效，而是帮你把“老板脑子里的东西”系统化

#### 建议的写法

- 把长段拆成短段
- 每一段只表达一个判断
- 少一点连续口播，多一点“结论句 + 解释句”

#### 推荐节奏

1. 张雪峰真正卖的是什么  
2. 为什么很多老板的生意离开本人就塌  
3. AI 为什么第一次让“系统化”变得可执行

## 4. 工具 / SOP 示例

### 个人网站制作 SOP

**内容定位**：教程类

#### 一句话目标

帮助完全不会代码的人，用 AI 从 0 到 1 做出能发布的个人网站。

#### 建议结构

1. 为什么内容创作者值得做个人网站  
2. 先找参考，不要从空白开始  
3. 先搭信息结构，不要先做动效  
4. 不要推倒重做，只做局部微调

#### 适合单独拆页的内容

- 参考网站清单
- 首屏结构模板
- 微调提示词

---

### 飞书 CLI

**内容定位**：工具教程

#### 一句话目标

解释为什么飞书 CLI 对 Agent 更友好，以及普通人怎么快速装起来。

#### 推荐结构

1. 先用一句话解释 GUI 和 CLI 的区别  
2. 再讲它为什么适合 AI  
3. 最后给安装入口和真实使用场景

#### 注意

- 安装命令不要直接埋在大段正文里
- 独立放一个“复制区”
- 把“它能做什么”和“为什么有价值”分开讲
